IT MANAGEMENT SYSTEM - CRUD ROUTING SECURITY REVIEW REPORT
=========================================================

Goal:
Check all modules for incorrect $crud_action handling that may break wrapper-action routing.

Context:
- Standard modules follow this pattern:
    • Wrapper files (create.php, edit.php, view.php, etc.) set $crud_action (e.g. 'create')
    • Then they require index.php
- Inside index.php, modules must NOT override $crud_action if it is already set.
- Correct pattern:
      if (!isset($crud_action)) { $crud_action = 'index'; }

Standard Pattern in it_settings/index.php:
-----------------------------------------
if (!isset($crud_action)) {
    $crud_action = $crud_action ?? 'index';
}
