Browser tools

Script Access What it does How to use
DBdesign.php Browser ER-style diagram generated from database.sql (Mermaid render, zoom, SVG/PNG export). Open DBdesign.php. Optional: ?format=mermaid, ?format=json. CLI: php scripts/DBdesign.php --mermaid or --json.
crud_tables.php Browser Lists every module folder and the first $crud_table = line found in index.php, with links to each module. Open in the browser to audit module-to-table mapping. CLI redirect: php scripts/crud_tables.php > crud_tables.html
update_all_created_at.php BrowserCLI After importing database.sql, sets every live row’s created_at to one timestamp (default 2026-01-01 00:00:01). Dry-run previews counts first. Open update_all_created_at.php while logged in. Optional CLI: php scripts/update_all_created_at.php --dry-run
detect_fk_dropdown_ui_risk_ui.php Browser Scans cross-tenant FK rows and module code that can cause duplicate dropdown options (e.g. Location id=1 and id=4 for the same tenant). Log in first. Open the UI, choose Scan mode, Company, Risk filter, and Output (table or JSON), then click Run scan. Equivalent CLI: php scripts/detect_fk_dropdown_ui_risk.php --company=4
detect_fk_dropdown_ui_risk.php BrowserCLI Same FK dropdown risk scan as above; in the browser it loads the UI file. Browser: same as detect_fk_dropdown_ui_risk_ui.php. CLI options: --company=N, --json, --data-only, --code-only, --help.

Security

Script Access What it does How to use
test_form_failed_save_display.php BrowserCLI Scans all module create forms for SQL-quoted re-display after failed saves (e.g. value="'USA'"); optional runtime POST tests per module. Log in first. Open test_form_failed_save_display.php, run static scan, optionally enable Runtime HTTP tests. CLI: php scripts/test_form_failed_save_display.php · runtime: --runtime with ITM_TEST_BASE_URL and ITM_TEST_COOKIE.
test_sql_injection.php Browser JSON sandbox to test SQL-injection signature detection and safe prepared-statement execution (GET/POST). Requires login and active company_id. GET/POST JSON with a test payload; POST must include a valid CSRF token. Use only in dev/staging — not a public endpoint.
sql_injection_matrix_test.php CLI Runs a fixed table of injection payloads through lib/sql_injection_detector.php and reports pass/fail. php scripts/sql_injection_matrix_test.php — exit code non-zero if any case fails.
check_sql_injection_coverage.php CLI Static scan: flags PHP files where queries may use user input without binding/sanitization. php scripts/check_sql_injection_coverage.php — run after CRUD/SQL changes (AGENTS.md requirement).
check_csrf_coverage.php CLI Static scan: finds POST handlers that change data without a known CSRF guard. php scripts/check_csrf_coverage.php
check_audit_logs_coverage.php CLI Static scan: verifies modules have audit trail coverage for INSERT/UPDATE/DELETE via itm_run_query, itm_log_audit, bulk delete helpers, or trg_{table}_audit_* triggers in database.sql. php scripts/check_audit_logs_coverage.php — exits 2 when a module mutates data with no audit path.
Optional: --module=rack_planner, --json
check_index_table_compliance.php CLI Static scan of module index.php list screens (plus module-local require/include PHP such as includes/partials/render.php). Enforces AGENTS.md index-table hooks and POST safety:
  • data-itm-db-import-endpoint on the list <table> (Excel import save-to-database via table-tools.js)
  • data-itm-actions-origin="1" on Actions <th> and <td> (layout engine / table_actions_position)
  • Known POST CSRF guards when index handlers mutate data (itm_require_post_csrf, cr_require_valid_csrf_token, etc.)
  • csrf_token hidden fields on POST forms
  • itm-actions-cell when an Actions column is present
Skips protection-zone modules and rack_planner. Grandfathers legacy gaps via scripts/data/index_table_compliance_baseline.txt (57 modules); CI fails only on new violations outside that list. 		php scripts/check_index_table_compliance.php — exit 1 on new violations (baseline-aware).
php scripts/check_index_table_compliance.php --strict — report all modules, ignore baseline.
php scripts/check_index_table_compliance.php --write-baseline — refresh grandfather list (use sparingly).
After fixing a module, remove its slug from scripts/data/index_table_compliance_baseline.txt.
smoke_test.sh CLI CI/local smoke runner (GitHub Actions .github/workflows/smoke.yml):
  1. php -l on every *.php file
  2. check_csrf_coverage.php
  3. check_sql_injection_coverage.php
  4. check_index_table_compliance.php
  5. check_ui_configuration_coverage.php (optional — skipped when SMOKE_SKIP_UI_CONFIG=1)
 bash scripts/smoke_test.sh — set SMOKE_SKIP_UI_CONFIG=1 to skip step 5 (UI layout audit) until legacy gaps are cleared.

Database

Script Access What it does How to use
analyze_database_health.php BrowserCLI Runs ANALYZE TABLE on every base table and lists per-table success/failure (avoids phpMyAdmin stopping on first error). Open analyze_database_health.php while logged in. Optional CLI: php scripts/analyze_database_health.php
check_database_sql_company_name_uniques.php BrowserCLI Audits database.sql: each company_id table needs 2 uniques (PRIMARY + UNIQUE on company_id + name or 3rd column). Open check_database_sql_company_name_uniques.php or php scripts/check_database_sql_company_name_uniques.php (exit 1 if any fail).
repair_table_from_schema.php CLI Rebuilds one InnoDB table from database.sql when metadata drift causes "doesn't exist in engine" errors. php scripts/repair_table_from_schema.php --table=table_name — destructive; backup first.
normalize_database_sql_created_at.php CLI Sets every seed created_at literal in database.sql INSERT rows to one timestamp (default 2026-01-01 00:00:01); leaves updated_at and other date columns unchanged. php scripts/normalize_database_sql_created_at.php — edits database.sql before import (not the live database).
