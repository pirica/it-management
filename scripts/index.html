Documentation

Script Access What it does How to use
api.php Browser HTML reference for JSON API endpoints (auth, CSRF, module import routes, request/response examples). Open in the browser after deploy. Use as the source of truth when calling /modules/*/index.php JSON handlers or shared APIs documented in the file.

Browser tools

Script Access What it does How to use
DBdesign.php Browser ER-style diagram generated from database.sql (Mermaid render, zoom, SVG/PNG export). Open DBdesign.php. Optional: ?format=mermaid, ?format=json. CLI: php scripts/DBdesign.php --mermaid or --json.
crud_tables.php Browser Lists every module folder and the first $crud_table = line found in index.php, with links to each module. Open in the browser to audit module-to-table mapping. CLI redirect: php scripts/crud_tables.php > crud_tables.html
update_all_created_at.php Browser + CLI After importing database.sql, sets every live row’s created_at to one timestamp (default 2026-01-01 00:00:01). Dry-run previews counts first. Open update_all_created_at.php while logged in. Optional CLI: php scripts/update_all_created_at.php --dry-run
detect_fk_dropdown_ui_risk_ui.php Browser Scans cross-tenant FK rows and module code that can cause duplicate dropdown options (e.g. Location id=1 and id=4 for the same tenant). Log in first. Open the UI, choose Scan mode, Company, Risk filter, and Output (table or JSON), then click Run scan. Equivalent CLI: php scripts/detect_fk_dropdown_ui_risk.php --company=4
detect_fk_dropdown_ui_risk.php Both Same FK dropdown risk scan as above; in the browser it loads the UI file. Browser: same as detect_fk_dropdown_ui_risk_ui.php. CLI options: --company=N, --json, --data-only, --code-only, --help.

Security

Script Access What it does How to use
test_sql_injection.php Browser JSON sandbox to test SQL-injection signature detection and safe prepared-statement execution (GET/POST). Requires login and active company_id. GET/POST JSON with a test payload; POST must include a valid CSRF token. Use only in dev/staging — not a public endpoint.
sql_injection_matrix_test.php CLI Runs a fixed table of injection payloads through lib/sql_injection_detector.php and reports pass/fail. php scripts/sql_injection_matrix_test.php — exit code non-zero if any case fails.
check_sql_injection_coverage.php CLI Static scan: flags PHP files where queries may use user input without binding/sanitization. php scripts/check_sql_injection_coverage.php — run after CRUD/SQL changes (AGENTS.md requirement).
check_csrf_coverage.php CLI Static scan: finds POST handlers that change data without a known CSRF guard. php scripts/check_csrf_coverage.php

Database

Script Access What it does How to use
analyze_database_health.php CLI Runs ANALYZE TABLE on every base table and prints per-table success/failure (avoids phpMyAdmin stopping on first error). php scripts/analyze_database_health.php — uses config/config.php DB connection.
repair_table_from_schema.php CLI Rebuilds one InnoDB table from database.sql when metadata drift causes "doesn't exist in engine" errors. php scripts/repair_table_from_schema.php --table=table_name — destructive; backup first.
normalize_database_sql_created_at.php CLI Sets every seed created_at literal in database.sql INSERT rows to one timestamp (default 2026-01-01 00:00:01); leaves updated_at and other date columns unchanged. php scripts/normalize_database_sql_created_at.php — sets seed created_at to 2026-01-01 00:00:01 on all tables with that column (INSERT VALUES + single-line replication SELECT; skips triggers).
