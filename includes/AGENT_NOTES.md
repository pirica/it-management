# AGENT_NOTES.md - Includes

## 1. Module Purpose
Contains shared PHP logic, helper functions, and visibility filters used across multiple modules.

## 2. Key Tables
- Primarily provides helper logic for **alerts**, **equipment**, and **audit_logs**.

## 3. Required Relationships
- Functions here often depend on `config/config.php`.

## 4. Business Rules (Critical for Agents)
- **Visibility Helpers**: `alerts_visibility.php` is mandatory for all alert-related queries.
- **Security Functions**: Use `itm_is_safe_identifier` for dynamic SQL identifiers.
- **Manufacturers CRUD template (`ui_config.php`):** Only `modules/manufacturers/` may host the live template. Other modules get **copied** PHP via `itm_materialize_manufacturers_crud_module_files()` / `itm_auto_create_module_scaffold()` — never `require __DIR__ . '/../manufacturers/…'`. New auto-scaffolded sidebar entries use **⚠️** (`itm_sidebar_auto_scaffolded_module_emoji()`). Legacy delegate stub folders are removed by QA cleanup (`itm_remove_manufacturers_template_scaffold_module_dirs()`).

## 7. File Structure

- **notes_visibility.php** — owner + `shared_with_json` filter for Notes module; `itm_notes_fetch_visible_by_id()` for single-record view/edit load; `itm_notes_private_images_dir()`, `itm_notes_normalize_image_filename()`, and `itm_notes_resolve_image_path()` confine note attachment ZIP downloads to `files/{company_id}/Private/{username}_{employee_id}/notes/`; `itm_notes_json_mutation_response()` standardises AJAX mutation JSON (404 when zero rows affected).
- **itm_api_json_response.php** — shared `itm_api_json_response()` and `itm_api_mutation_requires_rows()` for Org Chart, Rack Planner, and other single-step mutation endpoints. Switch-port **`update_port.php`** uses a manual zero-row `404` check before To IDF auto-sync (cannot call `itm_api_mutation_requires_rows()` mid-flow because that helper exits on success).
- **itm_maintenance_script_admin_gate.php** — `itm_enforce_maintenance_script_admin_browser()` restricts browser access to maintenance/security scripts to Admin users (CLI unchanged).
- **todo_visibility.php** — global/assigned/creator filter for Todo module.
- **delete_functions.php** — shared logic for complex deletions (e.g., equipment).
- **companies_view_redirect.php** — legacy company view redirect; guarded via `itm_script_entry_guard.php`.
- **get_ports.php** / **update_port.php** — switch port AJAX endpoints; guarded + shared helpers in **switch_port_api_helpers.php** (lookup maps and VLAN lists use `itm_mysqli_stmt_fetch_all_assoc()`); all JSON exits use `itm_api_json_response()` with `JSON_UNESCAPED_UNICODE`; prepared-statement reads use `itm_mysqli_stmt_fetch_assoc()` / `itm_mysqli_stmt_fetch_all_assoc()` (mysqlnd fallback). Tenant `$company_id` cast to `(int)` after `config.php`, from session only (HTTP `403` when missing; client payload `company_id` unset from `$input`/`$_POST`/`$_REQUEST`). **`update_port.php`** wraps `switch_ports` UPDATE plus To IDF auto-sync (`idf_ports` INSERT/UPDATE/DELETE) in a single transaction when `management_id` column exists; rolls back on zero-row update, IDF sync failure, or empty-position 422. Zero-row contract regression: `php scripts/verify_update_port_zero_row.php`.
- **switch_port_api_helpers.php** — shared lookup/VLAN helpers for port AJAX endpoints; tenant-scoped prepared queries use mysqlnd-safe fetch helpers.
- **itm_script_entry_guard.php** — `itm_skip_http_entry_unless_direct()`, `itm_skip_view_partial_unless_context()`, PHPUnit processing detection. **`ITM_HTTP_ENDPOINT_CONTRACT_TEST` is evaluated before `itm_is_phpunit_processing()`** so trusted CLI contract subprocesses (and `ItmScriptEntryGuardTest`) can include HTTP endpoints under CLI without the PHPUnit bare-include guard winning. **Threat model:** HTTP JSON endpoints normally no-op under CLI unless the request is the direct Apache entry script. Trusted regression subprocesses may define **`ITM_HTTP_ENDPOINT_CONTRACT_TEST`** before including an endpoint (for example `scripts/verify_update_port_zero_row.php`); that constant must **never** be derived from `$_GET`/`$_POST`/request headers or any other HTTP input — only hard-coded in vetted CLI scripts under `scripts/`.
- **switch_port_api_helpers.php** — shared lookup/VLAN helpers for port AJAX endpoints (avoids redeclare fatals during coverage).
- **itm_select_options_policy.php** — whitelist and blocked-table policy for `modules/select_options_api.php` quick-add inserts; includes **`license_types`** for License Management Type dropdown quick-add; **`companies`** is blocked (use Companies CRUD module instead).
- **itm_ui_action_labels.php** — canonical emoji-only labels for View/Edit/Delete/Back/Create/Save (`itm_ui_action_emoji()`, `itm_ui_action_title()`); NO MIXED contract patterns for `scripts/check_ui_action_emoji.php`. Loaded from `config/config.php`; JS mirror: `js/itm-ui-action-labels.js` (included from `header.php`).
- **itm_explorer_paths.php** — `explorer_normalize_relative_path()` and `explorer_extract_zip_safely()` shared by Explorer API/file serving.
- **itm_login_attempt_identifier.php** — `itm_normalize_login_attempt_identifier()` stores valid emails and simple usernames (`[a-zA-Z0-9_\-\.]`, max 64); redacts any other value, including strings with `@` that fail `FILTER_VALIDATE_EMAIL` (common when passwords are mistyped into the login field). Redacted values use `[redacted:{sha256-prefix}]` via `itm_login_attempt_identifier_redacted()`.
- **itm_employees_auth_sensitive_fields.php** — canonical Employees list/view column denylist (`password`, `vault_key_hash`, reset-token fields); `modules/employees/index.php` merges `itm_employees_auth_sensitive_field_names()` into list `$hiddenColumns`.
- **itm_vault_master_key.php** — `itm_vault_reencrypt_password_entries()` re-encrypts all `password_entries` for one employee during vault master key change; caller owns transaction COMMIT/ROLLBACK. Used by `user-config.php`.
- **itm_employees_hidden_accounts.php** — DB-only `employees.is_hidden` ensure helper, visible-only SQL predicate, and UI exclusion list; hidden rows are omitted from Employees index/list and blocked on view/edit/delete/import/clear-table.

- **itm_equipment_search.php** — equipment index list search JOIN + WHERE helpers matching scalar fields (dates, purchase cost, workstation/switch/printer columns, `active`, `photo_filename`) and create/edit FK labels (department code, supplier code, location code, rack code, assignee username, type/manufacturer/status, switch/workstation lookup names).
- **itm_employees_search.php** — Employees index list search helper; extends raw column LIKE with FK label JOIN columns (status, role, department name/code, location code, position name/description, manager name/username, etc.); includes `CONCAT(first_name, ' ', last_name)` when both name columns are visible; always searches `mobile_phone` even when hidden from list columns (ESA matrix).
- **itm_crud_fk_label_search.php** — shared flattened CRUD search helper; adds tenant-scoped `EXISTS` predicates so list search matches FK label tables (`itm_crud_fk_label_search_conditions()`). Wired into scaffold `modules/*/index.php` search blocks (except bespoke modules without search).
- **itm_todo_search.php** — Todo index search helper; extends title/description LIKE with category, department, and assignee label EXISTS for CSV `*_id` columns (`itm_todo_build_search_clause()`).
- **itm_employees_delete_dependencies.php** — `itm_employees_detach_delete_dependencies()` clears or nulls inbound links (`attempts`, `audit_logs`, `employee_companies`, `employee_system_access`, passwords/bookmarks, approvers, tickets reassignment, etc.) before admin employee delete; used by `modules/employees/delete_functions.php`.
- **itm_employee_employment_status.php** — login/dropdown eligibility via `employment_status_id` → `employee_statuses.name` = Active (`itm_employee_active_employment_status_join_sql()`, `itm_employee_employment_status_predicate_by_name_sql()`, `itm_employee_active_employment_status_predicate_sql()`, `itm_employee_on_leave_employment_status_predicate_sql()`, `itm_employee_resolve_employment_status_id_by_name()`, `itm_employee_count_by_employment_status_name()` for dashboard HR stat cards, `itm_employee_has_active_employment_status()`, `itm_employee_resolve_active_status_id()` for `register.php`). Replaces deprecated `employees.active`.
- **itm_active_sessions.php** — dashboard **Online now** metric: writable presence touch files under `sys_get_temp_dir()/itm-session-presence/{company_id}/{employee_id}.presence`; `itm_active_sessions_touch()` called from `config/config.php`; `itm_count_logged_in_users_for_company()` counts recent touches within `session.gc_maxlifetime`.
- **itm_api_rate_limit.php** — tier hourly limits; **Free** = unlimited, **no API key**, **session required** (`itm_api_resolve_rate_limit_row()` reads `$_SESSION['company_id']` + `$_SESSION['employee_id']` when no key); `itm_api_tier_requires_api_key()`; probe payload `itm_api_build_rate_limit_probe_payload()` (`employee_id`, `api_key_required`); `itm_api_enforce_rate_limit_or_exit()` for programmatic endpoints.
- **itm_password_reset.php** — `itm_password_reset_find_user_by_identifier()` (email or username), `itm_password_reset_store_token_for_employee()` (`DATE_ADD(NOW(), INTERVAL 24 HOUR)` expiry via `itm_password_reset_token_ttl_hours()`), `itm_password_reset_lookup_employee_by_token()` (hash + legacy plain `reset_token` fallback), `itm_password_reset_complete_for_employee()`; loaded from `config/config.php`.
- **itm_email.php** — tenant SMTP profiles (`email_smtp_configurations`), send log (`emails`), alert rules (`email_alert_rules`); `itm_send_email()`, `itm_email_get_default_smtp_config()`, `itm_email_dispatch_to_rule()`; `itm_email_build_transactional_html()` / `itm_email_apply_transactional_template()` wrap fragment HTML in login-style purple card layout (auto-applied unless `email_template => false` or body is already a full document); `itm_email_brand_name_for_template()` strips a leading gear from `app_name` before the template adds one in the header; loaded from `config/config.php`.

- **itm_company_module_access.php** — `has_module_access()`, `get_company_modules()`, `itm_list_all_modules_registry()`, `itm_ensure_registry_rows_for_module_slugs()`, `itm_has_module_access_bust_cache()`, `itm_sidebar_discovery_probe_cleanup()`, `itm_enforce_module_access_or_exit()`, registry sync/seed helpers; loaded from `config/config.php`. `has_module_access()` prefetches all `company_module_access` rows for the active company once per request (optional fourth arg `$forceRefresh`); `itm_set_company_module_access()` and registry inserts via `itm_ensure_registry_rows_for_module_slugs()` bust that cache after writes.

- **fk_dropdown_helpers.php** — tenant FK label resolution (`itm_fk_label_column_for_table()` prefers `name_type` for `employee_type`); business-key remap via `itm_fk_resolve_company_equivalent_id()`.

- **itm_system_status_cache.php** — `system_status` table cache get/save/refresh (`itm_system_status_cache_get($conn, $tabKey, $companyId)`, `itm_system_status_refresh_all()`); collectors for Monitoring, PHP Settings, and Database tab payloads; `cache_get()` scopes by `company_id`, uses `itm_mysqli_stmt_fetch_assoc()` (mysqlnd fallback), and routes execute failures through `itm_system_status_safe_stmt_execute()` (`errno` + correlation id in `error_log()`, no raw `mysqli_stmt_error()` text). `cache_save()` logs `json_encode` failures with `tab_key` and `company_id`.
- **itm_system_status_powershell.php** — Windows hardware runner: `shell_exec` availability, `.ps1` readability, `itm_system_status_run_powershell_action()` (hardware-only action allowlist + `[a-z0-9_]+` guard before loading `includes/{action}.ps1`; PHP/MySQL actions stay native).

Regression: `php scripts/verify_metadata_column_cache.php`.
- **database_sql_unique_audit.php** — parses `database.sql` CREATE TABLE blocks for tenant unique-key policy; `employee_companies` scope column is `employee_id` (not legacy `user_id`). Used by `php scripts/check_database_sql_company_name_uniques.php`.
